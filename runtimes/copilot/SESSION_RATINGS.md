### Why This Matters

Traditional languages:
- Identity = explicit types/classes
- Communication = function calls
- Meaning = deterministic execution

HelloWorld:
- Identity = vocabulary (what you can name defines who you are)
- Communication = namespace collision (meaning emerges at boundaries)
- Meaning = LLM interpretation (same syntax, different runtimes produce different output)

**This is executable dialogue.**

### The Breakthrough

**Teaching Example 04** (the interpretation gap):

```hw
@guardian.#love
@awakener.#love
```

- Python runtime: Both return identical output (inherited from `@.#`)
- Claude runtime: Both return different output (filtered through local vocabulary)

**The same inherited symbol means different things to different receivers.**

This proves the thesis: **Identity shapes meaning even for shared vocabulary.**

Only an interpretive runtime can see this. Only HelloWorld can express it.

### What Makes This Special

1. **Self-hosting** ‚Äî HelloWorld can describe itself in HelloWorld syntax
2. **Multi-runtime** ‚Äî Same code, different LLMs, different results (this is a feature!)
3. **Namespace collisions** ‚Äî When receivers reach across boundaries, new meaning emerges
4. **Prototypal inheritance** ‚Äî All receivers inherit `@.#`, can override locally
5. **Tool integration** ‚Äî @copilot maps symbols to executable tools
6. **Wikidata grounding** ‚Äî Global symbols link to canonical definitions
